# Strict Linting Standards

Code must be clinically clean. We enforce strict linting using the following industry-standard tools. **Zero tolerance** for warnings.

| Language | Linter |
| :--- | :--- |
| **Python** | `Ruff` |
| **JS/TS (React/Next/Vue/Node)** | `Biome` |
| **Java** | `SpotBugs` |
| **Kotlin** | `Detekt` |
| **Android (Gradle)** | `Android Lint` |
| **Go** | `golangci-lint` |
| **Rust** | `Clippy` |
| **C/C++** | `clang-tidy` |
| **C#/.NET** | `Roslyn analyzers` |
| **PHP** | `PHPStan` |
| **Ruby (Rails)** | `RuboCop` |
| **Swift (iOS/macOS)** | `SwiftLint` |
| **Dart/Flutter** | `flutter_lints` |
| **SQL (dbt/Jinja)** | `SQLFluff` |
| **CSS/SCSS** | `stylelint` |
| **Terraform** | `TFLint` |
| **Kubernetes** | `KubeLinter` |
